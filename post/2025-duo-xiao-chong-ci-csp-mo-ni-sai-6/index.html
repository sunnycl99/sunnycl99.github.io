在概率题解中，容斥原理被用于计算前 nnn 个数和等于后 nnn 个数和的概率，具体来说，是计算 2n2n2n 个在 [0,m][0, m][0,m] 范围内均匀随机的整数的和等于 nmnmnm 的方案数。以下是容斥部分的详细解释。

问题背景

  • 我们有两个集合，各包含 nnn 个在 [0,m][0, m][0,m] 范围内均匀随机的整数。
  • 想要求前 nnn 个数的和等于后 nnn 个数的和的概率，即 P(S=T)P(S = T)P(S=T)。
  • 通过变换（将后 nnn 个数乘 −1-1−1 再加 mmm），问题转化为求 2n2n2n 个在 [0,m][0, m][0,m] 范围内的整数的和等于 nmnmnm 的概率。

为什么需要容斥？

  • 直接计算 2n2n2n 个数的和等于 nmnmnm 且每个数在 [0,m][0, m][0,m] 范围内的方案数很困难，因为每个数有上界 mmm。
  • 容斥原理允许我们先计算无上界限制时的方案数，然后减去那些至少有一个数超过 mmm 的方案数，从而得到满足条件的方案数。

容斥原理的作用

容斥原理在这里系统地处理了每个数的上界限制，通过减去“超过上界”的方案数来校正总方案数。这种方法是组合数学中处理约束问题的标准技巧，类似于 OI-wiki 上容斥原理在不定方程非负整数解问题中的应用。

在容斥原理中，符号项 (−1)i(-1)^i(−1)i 起着关键作用，它确保了在计算满足多个条件的方案数时，能够正确地处理重叠部分，避免多算或少算。下面我将详细解释符号项的作用，特别是结合概率题解中的容斥公式。

容斥原理的基本思想

容斥原理用于计算满足一系列条件的方案数，当这些条件有重叠时，直接计算变得复杂。基本思路是：

  • 首先，计算所有可能方案的总数（无限制情况）。
  • 然后，减去不满足每个条件的方案数。
  • 但由于不满足条件的方案可能有重叠（例如，同时不满足多个条件），我们需要通过交替加减来校正这些重叠部分。

符号项 (−1)i(-1)^i(−1)i 正是实现这种交替加减的关键：当考虑 iii 个条件不满足时，我们乘以 (−1)i(-1)^i(−1)i，使得在求和过程中，每个方案被精确地计算一次。

在概率题解中的具体应用

在题解中，我们要求 2n2n2n 个在 [0,m][0, m][0,m] 范围内的整数的和等于 nmnmnm 的方案数。容斥公式为：
N=∑i=02n(−1)i(2ni)(nm−(m+1)i+2n−12n−1) N = \sum_{i=0}^{2n} (-1)^i \binom{2n}{i} \binom{nm - (m+1)i + 2n - 1}{2n - 1} N=∑i=02n​(−1)i(i2n​)(2n−1nm−(m+1)i+2n−1​)
其中：

  • (2ni)\binom{2n}{i}(i2n​) 表示从 2n2n2n 个数中选择 iii 个数，这些数超过 mmm（即至少为 m+1m+1m+1）。
  • (nm−(m+1)i+2n−12n−1)\binom{nm - (m+1)i + 2n - 1}{2n - 1}(2n−1nm−(m+1)i+2n−1​) 表示在给这 iii 个数每个分配 m+1m+1m+1 后，剩余总和 nm−(m+1)inm - (m+1)inm−(m+1)i 分配给 2n2n2n 个非负整数的方案数（无上界限制）。
  • 符号项 (−1)i(-1)^i(−1)i 用于调整计数，确保最终只计入所有数都在 [0,m][0, m][0,m] 内的方案。

符号项 (−1)i(-1)^i(−1)i 的作用详解

  1. 当 i=0i=0i=0：

    • 项为 (−1)0(2n0)(nm+2n−12n−1)(-1)^0 \binom{2n}{0} \binom{nm + 2n - 1}{2n - 1}(−1)0(02n​)(2n−1nm+2n−1​)，即所有无限制方案数。这包括了所有数都在 [0,m][0, m][0,m] 内的方案，也包括了有些数超过 mmm 的方案。但此时我们还没有排除任何方案，所以符号为正。

  2. 当 i=1i=1i=1：

    • 项为 (−1)1(2n1)(nm−(m+1)+2n−12n−1)(-1)^1 \binom{2n}{1} \binom{nm - (m+1) + 2n - 1}{2n - 1}(−1)1(12n​)(2n−1nm−(m+1)+2n−1​)，即减去那些至少有一个数超过 mmm 的方案数。但这样减多了，因为如果一个方案有两个数超过 mmm，它会被减去两次（一次在 i=1i=1i=1 时针对第一个数，一次针对第二个数）。

  3. 当 i=2i=2i=2：

    • 项为 (−1)2(2n2)(nm−2(m+1)+2n−12n−1)(-1)^2 \binom{2n}{2} \binom{nm - 2(m+1) + 2n - 1}{2n - 1}(−1)2(22n​)(2n−1nm−2(m+1)+2n−1​)，即加回那些至少有两个数超过 mmm 的方案数。这补偿了在 i=1i=1i=1 时多减的部分。

  4. 依此类推：

    • 对于每个 iii，符号项 (−1)i(-1)^i(−1)i 使得我们交替地加减方案数。这样，如果一个方案恰好有 kkk 个数超过 mmm，它会在 i=0i=0i=0 到 i=ki=ki=k 项中被计算，但通过符号项，它的净贡献为：
      ∑j=0k(−1)j(kj)=(1−1)k=0当 k≥1 \sum_{j=0}^{k} (-1)^j \binom{k}{j} = (1-1)^k = 0 \quad \text{当 } k \geq 1 ∑j=0k​(−1)j(jk​)=(1−1)k=0当 k≥1
    • 这意味着，任何至少有一个数超过 mmm 的方案在总和中净贡献为 0，而所有数都在 [0,m][0, m][0,m] 内的方案只被 i=0i=0i=0 项计算一次（因为当 i=0i=0i=0 时，符号为正，且没有排除任何数），从而被正确计入。

为什么需要符号项？

如果没有符号项，我们只是简单地从无限制方案中减去至少一个数超过 mmm 的方案，但会多减那些多个数超过 mmm 的方案。符号项通过交替加减，确保了每个方案只被计算一次，且只计入所有条件都满足的方案。这是一种经典的包含-排除（inclusion-exclusion）方法。

在概率题解中，这使我们能准确计算前 nnn 个数和等于后 nnn 个数和的概率，进而得到前 nnn 个数和大于后 nnn 个数和的概率。

#include<bits/stdc++.h>
#define int long long
#define fo(i,a,b) for(int i=a;i<=b;i++)
#define fd(i,a,b) for(int i=a;i>=b;i--)
using namespace std;
const int N=4000020;
int n,m,mo;
int fpow(int x,int y)
{
    int ans=1;
    while(y)
    {
        if(y%2)ans=ans*x%mo;
        x=x*x%mo;
        y/=2;
    }
    return ans;
}
int fac[N],caf[N];

int C(int n,int m)
{
    if(m>n)return 0;
    return fac[n]*caf[m]%mo*caf[n-m]%mo;
}
signed main()
{
    // freopen("pr.in","r",stdin);
    // freopen("pr.out","w",stdout);
    int T;
    cin>>mo>>T;
    fac[0]=caf[0]=1;
    fo(i,1,N-20)fac[i]=fac[i-1]*i%mo;
    caf[N-20]=fpow(fac[N-20],mo-2);
    fd(i,N-21,1)caf[i]=caf[i+1]*(i+1)%mo;
    while(T--)
    {
        scanf("%lld%lld",&n,&m);
        int ans=0;
        fo(i,0,min(n*m/(m+1),2*n))
            ans+=(fpow(-1,i)*C(2*n,i)%mo*C(n*m-(m+1)*i+2*n-1,2*n-1)%mo+mo)%mo,
            ans%=mo;
        ans=ans*fpow(fpow(m+1,2*n),mo-2)%mo;
        ans=(1-ans+mo)%mo*fpow(2,mo-2)%mo;
        printf("%lld\n",ans);
    }
    return 0;   
}
